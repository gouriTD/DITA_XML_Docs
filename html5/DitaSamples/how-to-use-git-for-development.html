How to use Git for Development?

Here we explore how Git is used for development.

Ensure Git is successfully installed on your systems.
Basic Git Workflow

Git follows a simple workflow for tracking changes in a project:

  1. Initialize a repository
  2. Stage files
  3. Commit changes
  4. Check repository status
  1. Initialize a repository

    Navigate to your project folder and initialize Git:

    git init

    This command creates a new Git repository in the selected directory.

  2. Stage files

    Add the files in the staging area before committing

    Add a specific file:

    git add <filename>

    Add multiple files in the directory: 

    git add .
  3. Commit changes

    Save the staged changes with a descriptive message.

    git commit -m <your descriptive message>
  4. Check repository status

    View the current state of repository:

    git status
The repository is initialized and tracked by Git.

Let’s try to understand how to track a project using Git with an example.

Suppose you have a project folder named testProject.

  1. Navigate to project folder:
    cd testProject
  2. Initialize the repository:
    git init
  3. Create a new file:
    touch testFile.txt
  4. Stage the newly created file:
    git add testFile.txt
  5. Commit the staged file:
    git commit -m “test file added”
  6. Check the repository status:
    git status
